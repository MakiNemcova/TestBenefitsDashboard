# Bug Reports Submission

This repository contains the results of exploratory testing and automated test coverage for the Benefits Dashboard application.

- The goal of this submission is to demonstrate:
Test design and prioritization
Ability to identify functional and contract-level defects
Basic UI and API automation using Playwright

It includes:

- **API-bugs.md** – all API-related issues with steps to reproduce.
- **UI-bugs.md** – all User Interface issues with steps to reproduce.

Each bug includes:
- Steps to reproduce
- Expected result
- Actual result

## Test Strategy

Focus was placed on authentication, authorization, and employee data flows because these areas carry the highest business and security risk.

Testing was performed using a combination of:

Exploratory testing (UI + API)
Negative scenario testing (authentication, validation, edge cases)
Targeted automation for regression coverage

Focus areas:

Authentication & authorization flows
API contract consistency
Data correctness between API and UI
Input validation and error handling

## Key Findings

- Critical / High-Impact Issues
Dashboard remains accessible after logout via browser back button
Protected routes accessible via direct URL after logout
Application state is not synchronized across multiple tabs (stale session)
Unauthorized access to "Add Employee" functionality without authentication

These issues represent security and access control risks

- Functional Issues
First Name and Last Name are swapped in the UI
Missing validation feedback in UI when API returns an error
No confirmation message after deleting an employee

These issues affect data correctness and user experience

- API Contract Issues

Username marked as required in documentation but auto-generated by API
Id inconsistently documented as optional in PUT request
Inconsistent naming: dependants (API) vs dependents (UI)
Required fields allow empty values due to minLength: 0
GET after delete returns 200 OK with empty body

These issues may cause integration problems for API consumers

- UX / Improvement Suggestions
Missing currency and time unit (e.g., yearly/monthly) for salary fields
Long IDs reduce readability (wrapping across lines)
Delete action lacks clarity (icon-only interaction)
No sorting or filtering options in employee table
Missing application version/build information

## Setup

1. Install dependencies:
npm install

2. Create .env
BASE_URL=https://your-url
API_USER=your-username
API_PASSWORD=your-password
UI_USER=your-username
UI_PASSWORD=your-password

3. Run all tests
npx playwright test

4. The following test is intentionally failing as it reveals a real defect in the application:
Test:
Scenario 1: Verify correct row by unique values

Issue:
First Name and Last Name values are swapped in the UI.

This test is included to demonstrate how automated tests can detect real defects in the application.

5. Run only API tests
npx playwright test tests/api

6. Run only UI tests
npx playwright test tests/ui

7. Run tests in headed mode (browser visible)
npx playwright test --headed

8. Technologies Used
Playwright
JavaScript / Node.js

## Known Limitations
Tests are not fully isolated (shared test account)
Parallel execution may cause failures
Sequential execution is recommended (workers: 1)
